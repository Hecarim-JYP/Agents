# 배포 준비

아래 체크리스트를 순서대로 수행하고, 각 항목을 통과(✅)/실패(❌)/해당없음(–)으로 **표로 보고**한다. **실패 항목이 있으면 배포 불가로 판정**하고 해결을 제안한다. 실제 배포 실행은 사용자 승인 후에만.

1. **테스트** — `npm test`(또는 pytest) 전체 실행. 실패 시 여기서 중단하고 보고.
2. **빌드** — 프로덕션 빌드(`tsc`, `vite build` 등) 성공 확인.
3. **의존성 취약점** — `npm audit` 실행, high 이상 취약점 보고 (불가피하면 사유 기록).
4. **마이그레이션** — `migrations/`의 미적용 파일 확인(적용 기록과 대조), 적용 순서와 멱등성 확인 (`~/.claude/jyp/conventions/migration.md`).
5. **환경변수 대조** — 이번 변경에서 추가된 환경변수가 `.env.example`과 운영 환경에 반영됐는지 확인.
6. **보안 체크리스트** — `~/.claude/jyp/conventions/ops.md` 5절의 9항목 점검 (첫 배포는 전 항목, 이후는 변경 영역만).
7. **작업 정리 상태** — changelog가 최신인지 확인. 안 되어 있으면 `/work-log` 먼저 수행을 제안.
8. **브랜치/태그** — main 병합 상태 확인, 배포 태그 이름 제안 (예: `v1.2.0` 또는 `deploy-YYYY-MM-DD`).
