# 통합 테스트 (정적 추적, paper execution)

앱을 실행하지 않고 **실제 코드를 읽어** 핵심 프로세스의 계층 간 계약을 검증한다.

## 방법론

- 검증 대상마다 **구체적 가상 시나리오**(입력값·권한·상태 명시)를 세우고, 진입점부터 저장소/외부 경계까지 각 계층 경계의 데이터 계약을 따라간다 (예: 화면 → API 훅 → 컨트롤러 → 서비스 → 쿼리 → DB → 응답 → 화면 반영).
- 추측 금지 — **`파일경로:라인` + 스니펫 근거**가 있는 것만 보고한다.

## 절차

1. **인벤토리 구축** — 프로젝트의 모듈/기능별 핵심 플로우(CRUD, 인증·권한, 횡단 관심사) 목록을 만들어 사용자에게 확인받는다. 프로젝트 CLAUDE.md에 인벤토리가 이미 있으면 그것을 기준으로.
2. **계약 체크리스트 준비** — 프로젝트 CLAUDE.md의 규칙·Critical Pitfalls를 경계별 검증 기준으로 삼는다.
3. **읽기 전용 에이전트 fan-out** — 플로우 단위로 병렬 분담. ⚠ 추적 에이전트는 **반드시 읽기 전용** — 코드 수정 금지, 발견은 보고만. 실제 수정은 사용자 승인 후 메인 세션에서 직접.
4. **라운드 분할** — 범위가 크면 여러 라운드로 나누고, 각 라운드 결과를 보고한 뒤 다음 진행.
5. **백로그 반영** — 발견된 이슈는 심각도·근거와 함께 `docs/REFACTORING_BACKLOG.md`에 추가.

## 출력 형식

- 플로우별: `시나리오 입력 → 거치는 함수/파일:라인 → 계약 검증(✅/⚠️/❌)`
- 발견 항목: 심각도(High/Med/Low) + 근거 + 수정 제안
- 마지막에 **① 확인된 버그 ② 의심(추가 확인 필요) ③ 정상 확인** 3분류 요약

## 범위 한계 (반드시 고지)

정적 분석이라 런타임 동작·실제 DB 데이터·외부 시스템 실응답·동시성·브라우저 렌더링은 검증 불가. 실동작 검증이 필요하면 서버 기동 스모크 또는 E2E로 에스컬레이션 제안.
