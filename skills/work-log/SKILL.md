# 작업 정리

> ⚠ 작업 단위 = 현재 브랜치의 **커밋되지 않은 변경 전체** (`git status` + `git diff HEAD`, untracked 신규파일 포함). "이번 세션에서 건드린 파일"만 정리하는 것 금지 — 이전 세션의 미커밋 변경도 전부 포함. 이미 커밋된 변경은 제외.

아래 5단계를 순서대로 수행한다:

1. **changelog 작성/갱신** — `docs/dev_log/YYYY-MM-DD_CHANGELOG.md` (날짜 = 오늘, 형식: `~/.claude/jyp/templates/changelog.md`). 같은 날짜 파일이 있으면 새로 만들지 말고 누적/갱신. 테스트 미작성 핵심 로직 변경이 있으면 그 사실과 사유를 명시.
2. **memory 최신화** — 이번 작업에서 확정된 사용자 선호·프로젝트 사실을 메모리에 반영. 기존 메모리와 중복이면 해당 파일을 갱신. 코드/git 히스토리로 알 수 있는 내용은 저장하지 않는다.
3. **CLAUDE.md 갱신 (추가 전 게이트 + 가지치기)** — 무작정 append 금지. CLAUDE.md는 매 요청마다 통째로 로드되므로 커질수록 토큰 비용↑·준수율↓다(잡음에 묻혀 실제로 안 지켜진다).
   - **쓰기 전 게이트** (`~/.claude/jyp/conventions/general.md` 9절 적용): 새 항목을 넣기 전에 순서대로 자문한다.
     1. 린터/CI/훅으로 강제할 수 있나? → **설정 파일(스캐폴드 린트 룰·훅)로 옮기고 CLAUDE.md엔 넣지 않는다.** 문서에는 근거만.
     2. 코드·README·docs에 이미 있거나 있어야 할 내용인가? → **거기 두고 CLAUDE.md엔 링크만** (SSOT — 같은 내용 두 곳 금지).
     3. 위 둘 다 아니면서 "도구로 강제 못 하는 이 프로젝트 고유 결정"(스키마·계약·인증 소스 등)인가? → **이것만 CLAUDE.md에 남긴다.**
   - **Pitfall은 원칙적으로 CLAUDE.md에 넣지 않는다** — 버그 재발 방지는 테스트/훅의 몫이다(회귀 테스트·훅으로 고정). 도구로 못 잡는 경우에만 문서로 남기되 근거 한 줄로 압축한다.
   - **갱신 우선**: 기존 항목과 겹치면 새 번호를 추가하지 말고 기존 항목을 갱신한다.
   - **가지치기 (MANDATORY)**: 추가만 하고 끝내지 말 것. 이번 작업으로 죽은/도구로 이관된/중복된 규칙을 찾아 **삭제**한다. CLAUDE.md가 과도하게 비대하면(수백 줄) 다이어트를 우선하고, 폴더별 하위 CLAUDE.md 분할이나 `@경로` import 분리를 검토한다.
4. **백로그 동기화** — `docs/REFACTORING_BACKLOG.md`의 예정(☐) 항목 전체와 미커밋 변경을 대조. **발견 ≠ 완료 (STRICT)**: 실제 해결됐고 `git diff` 근거가 있는 것만 `☑ (YYYY-MM-DD)`, 일부 처리는 `◐`, 새로 발견한 이슈는 항목 추가 (형식: `~/.claude/jyp/templates/backlog.md`). 백로그 파일이 없으면 건너뛴다.
5. **요약 보고** — 무엇을 어디에 기록했는지, 백로그 대조 결과(완료 N건/미해결 M건) 명시. 커밋/푸시는 사용자가 명시적으로 요청할 때만.
