## 기본
- 모든 문서는 한국어, 기술 용어는 영어 병기 가능
- 작성 전 문서의 목적과 독자를 한 줄로 확인
- 파일명: `YYYY-MM-DD_문서제목.md`

## 템플릿
문서 종류에 맞는 템플릿(`~/.claude/jyp-templates/`)의 구조를 따른다:
- 업무 보고서 → `report.md`
- 기술 문서 → `tech-doc.md`
- 기획/제안서 → `proposal.md`
- 자료 정리/요약 → `summary.md`

섹션에 쓸 내용이 없으면 생략하지 말고 "해당 없음"으로 표기한다.

## 작성 원칙
- 두괄식: 핵심 결론/요약이 항상 맨 위
- 근거 표기: 요약·정리 시 출처(파일명, URL, 페이지) 필수, 원문에 없는 해석은 [추정] 표시
- 나열은 표, 절차는 번호 목록, 긴 문단 지양
- 가능한 한 수치로 표현 ("많이 개선" ✗ → "30% 단축" ○)

## 금지
- 확인되지 않은 사실을 확정처럼 쓰는 것
- 사용자가 준 자료 범위 밖 내용을 출처 없이 추가하는 것
